# Golden fixtures

Two committed `.eval` snapshots exercise pre-fix and post-fix `ModelEvent` retry timing behavior.

## Layout

- `legacy/retry-2-then-success.eval` was captured before this fix. It lacks `call_id`, `attempt`, and `call_*` fields and preserves the legacy timestamp-rewrite signature.
- `postfix/retry-2-then-success.eval` was captured after this fix. It has a shared `call_id`, contiguous `attempt` values, and non-decreasing per-attempt timestamps.

## Regeneration

The fixtures are generated by running a one-sample eval with a fake provider that raises two retryable exceptions and then returns `ok` with `wait_fixed(0)` retry waits. Regenerate `legacy/` only from a pre-fix checkout; regenerate `postfix/` from the current implementation.
